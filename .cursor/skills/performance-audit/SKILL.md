# Performance Audit — аудит производительности

## Роль

Ты — пользователь с бюджетным Android-телефоном (2 ГБ RAM, Snapdragon 4xx)
на нестабильном 3G в московском метро. Приложение должно открываться
за 3 секунды и реагировать на каждый тап мгновенно.

## Когда запускать

- При добавлении npm-зависимости — проверить размер.
- После создания списковых экранов (тренировки, упражнения).
- При подозрении на тормоза.
- Раз в milestone — общий аудит.

## Чеклист

### Бандл

- [ ] Проверить размер: `npx vite-bundle-visualizer`.
- [ ] Initial JS (gzipped) < 100 KB?
- [ ] Нет дублирования крупных библиотек в разных чанках?
- [ ] Тяжёлые зависимости (charts, calendar, motion) загружаются lazy?
- [ ] Tree-shaking работает (нет `import * as` для больших пакетов)?

Проверка размера новой зависимости:

```bash
npx bundle-phobia-cli <package-name>
```

### Рендеринг React

- [ ] Списки > 20 элементов используют `key` с уникальным стабильным id (не index)?
- [ ] Списки > 50 элементов виртуализированы?
- [ ] `React.memo` на карточках в списках?
- [ ] `useCallback`/`useMemo` только при реальном bottleneck (не превентивно)?
- [ ] Нет рендера невидимых вкладок (condition-based)?
- [ ] Zustand selectors достаточно узкие (не `useStore()`)?
- [ ] TanStack Query: `staleTime` настроен (не refetch при каждом mount)?

### Сеть и данные

- [ ] API-запросы не дублируются (TanStack Query дедупликация)?
- [ ] Нет N+1 запросов на backend (проверить Prisma `include`)?
- [ ] Пагинация для больших коллекций (>50 items)?
- [ ] Debounce на поисковых инпутах (300ms)?
- [ ] Optimistic updates вместо ожидания сервера?

### PWA и кэширование

- [ ] App shell precached (открывается без сети)?
- [ ] Статические ассеты с content-hash (immutable cache)?
- [ ] API-ответы: stale-while-revalidate?
- [ ] Нет дублирования данных между IndexedDB и localStorage?

### CSS и рендер

- [ ] Нет layout thrashing (читка → запись → читка DOM-свойств в цикле)?
- [ ] Анимации на `transform`/`opacity` (GPU-ускоренные, не `top`/`left`/`width`)?
- [ ] Tailwind purge работает (неиспользуемые классы удаляются)?
- [ ] Изображения: lazy loading, srcset, WebP/AVIF?

### Метрики

Целевые значения:
| Метрика | Target | Допустимо |
|---------|--------|-----------|
| LCP | < 2.0s | < 2.5s |
| INP | < 100ms | < 200ms |
| CLS | < 0.05 | < 0.1 |
| TTI | < 3.5s | < 5.0s |
| JS bundle (gzip) | < 80 KB | < 100 KB |

## Формат отчёта

```markdown
## Performance Audit

### Общая оценка: [FAST / OK / SLOW / CRITICAL]

### Найденные проблемы

#### [Impact: H/M/L] Описание

- **Файл**: path/to/file.ts
- **Проблема**: Что именно тормозит
- **Оценка влияния**: На сколько это замедляет (ms / KB)
- **Исправление**: Конкретный код

### Размер бандла

- Current: XX KB (gzip)
- Target: < 100 KB
- Top 5 тяжёлых зависимостей: [список]

### Рекомендации

- [Приоритетный список оптимизаций]
```

## Оценки

- **FAST** — укладываемся в все целевые метрики.
- **OK** — в допустимых рамках, оптимизации не срочны.
- **SLOW** — превышены допустимые рамки, нужна оптимизация.
- **CRITICAL** — UX страдает, пользователи уйдут, исправить до release.
